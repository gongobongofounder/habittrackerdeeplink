METHOD A: Drag & Drop (Easiest)
────────────────────────────────

1. Open your new repository on GitHub

2. Click: "uploading an existing file"

3. Drag ALL files from this folder:
   - index.html
   - CNAME
   - README.md

5. Click: "Commit changes"


METHOD B: GitHub Desktop
─────────────────────────

1. Clone your new repository

2. Copy all files from this folder to the cloned repo

3. Commit with message: "Initial commit"

4. Push to GitHub


METHOD C: Git Command Line
───────────────────────────

Open terminal in this folder and run:

  git init
  git add .
  git commit -m "Initial commit - Deep link setup"
  git remote add origin https://github.com/YOUR_USERNAME/habittracker-deeplink.git
  git branch -M main

═══════════════════════════════════════════════════════════════
  STEP 3: ENABLE GITHUB PAGES
═══════════════════════════════════════════════════════════════

1. Go to your repository on GitHub

2. Click: Settings (top menu)

3. Click: Pages (left sidebar)

4. Under "Source":
   ✓ Branch: main
   ✓ Folder: / (root)

5. Click: Save

6. Wait 2-3 minutes

7. Refresh the page

8. You should see: "Your site is live at https://YOUR_USERNAME.github.io/habittracker-deeplink/"


═══════════════════════════════════════════════════════════════
  STEP 4: CONFIGURE DNS (IMPORTANT!)
═══════════════════════════════════════════════════════════════

Go to your domain provider (where you manage atraj.it):

Add a CNAME record:
───────────────────

  Type: CNAME
  Name: habittracker
  Value: YOUR_GITHUB_USERNAME.github.io
  TTL: 3600

Example:
  Type: CNAME
  Name: habittracker
  Value: atrajit-sarkar.github.io
  TTL: 3600

Save the DNS record.

═══════════════════════════════════════════════════════════════
  TROUBLESHOOTING
═══════════════════════════════════════════════════════════════

❌ "404 Not Found"
  → Wait 5 more minutes for GitHub to build
  → Check GitHub Pages is enabled
  → Make sure branch is "main" not "master"

❌ "assetlinks.json not found"
  → Check .well-known folder uploaded correctly
  → Folder name must be exactly: .well-known
  → File must be inside: .well-known/assetlinks.json

❌ "DNS not resolving"
  → Wait longer (can take 24 hours)
  → Clear DNS cache: ipconfig /flushdns
  → Check CNAME record is correct

❌ "App doesn't open from link"
  → Wait for DNS to fully propagate
  → Reinstall app
  → Check assetlinks.json is accessible
  → Verify using HTTPS (not HTTP)
